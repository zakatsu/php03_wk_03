# ①課題名
ManiArc mini03（熱量アーカイブアプリ）

## ②課題内容（どんな作品か）
- 推し活・スポーツ・ゲームなど、自分が「何に、いつから、どのような形で夢中になっているか」を宣言できるアプリです。
- 登録した内容をMySQLデータベースに保存し、「熱量パスポート」として一覧表示します。継続年数の集計、ジャンル別の絞り込み、登録内容の更新・削除もできます。
- 今回の課題要件に合わせて、登録時は `INSERT文`、表示時は `SELECT文`、更新時は `UPDATE文`、削除時は `DELETE文` を使っています。

## ⑤工夫した点・こだわった点
- 単なる入力一覧ではなく、一つひとつの記録を「熱量パスポート」として見せるデザインにしました。
- 選択したジャンルに応じて「愛の形」の説明文が変わるようにし、入力内容をイメージしやすくしました。
- DB接続は共通ファイルにまとめ、`write.php` ではフォームの入力値を安全にDBへ保存するためにPDOを使って `INSERT文` を実行しています。
- `read.php` では `SELECT` でDBから記録を取得し、アーカイブ画面に表示しています。
- `edit.php` では対象名・愛の形・ひとことを更新できるようにし、PDOのプリペアドステートメントで `UPDATE文` を実行しています。
- `delete.php` を追加し、アーカイブ画面から自分のパスポートだけを削除できるようにしました。削除時は `DELETE文` を実行しています。
- 公開URLでも他の人の登録内容がそのまま見えないように、ブラウザごとの匿名IDを使って表示対象を分けています。
- DB接続に必要なパスワードなどの情報は、GitHubに含めないように管理しています。
- サーバー側でも必須項目、文字数、年の範囲、選択肢を検証し、表示時にはHTMLエスケープを行っています。
- アーカイブ全体の継続年数を集計し、ジャンル別にパスポートを絞り込めるようにしました。

## ⑥難しかった点・次回トライしたいこと（又は機能）
- CSV保存からDB保存へ切り替えた結果、さくらサーバーではローカルXAMPPとDB接続情報が異なるため、何がローカル環境で、何がサーバー上で実行するものか、構成を理解することに意識を向けました。
- 更新・削除でも、他のブラウザで登録されたデータを操作できないように、`id` だけではなくCookieの匿名IDも条件に入れました。
- 公開URLとして提出する以上、アクセスした人全員に登録内容が見えてしまう点が課題でした。今回はログイン機能までは実装せず、ブラウザごとの匿名IDをCookieに保存し、DB登録時にも同じIDを保存することで、同じブラウザで登録した内容だけを表示する形にしました。
- 今後はログイン機能を追加し、CookieではなくユーザーIDで登録内容を管理できるようにしたいです。
